codex-provider-switch Proxy Mode Architecture

Codex keeps one local base_url. cps proxy reads the active provider on every request and injects the matching API key.

Codex sees one endpoint

  • base_url stays on localhost
  • auth.json stores only a placeholder key
  • existing Codex processes reuse the same local URL

Proxy resolves each request

  • loads providers.json per request
  • chooses the current active provider
  • injects the real upstream API key

Switching is fast

  • cps use only updates activeProvider
  • new requests use the new provider
  • active streams finish on their original provider
